### Layer Responsibilities

ğŸ¨ **Presentation Layer**  
Handles UI rendering, user interactions, and state management. Built with Flutter widgets and managed by Flutter Bloc/Cubit for predictable, reactive state updates. Each screen is isolated and communicates with domain layer through use cases.

ğŸ§  **Domain Layer**  
Contains pure business logic, entity definitions, and repository contracts. Platform-agnostic and framework-independent, this layer defines *what* the app does without *how* it's implemented. All business rules live here.

ğŸ’¾ **Data Layer**  
Implements repository contracts, manages data sources (REST APIs via Dio, local storage via Hive), and handles data transformation between DTOs and domain entities. Provides offline-first capabilities and network resilience.
